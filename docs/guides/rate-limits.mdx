---
title: "Rate Limits & Usage"
description: "How rate limiting and credit quotas work."
---

Unwrap uses a two-dimensional rate limiting system: **credits** (monthly quota)
and **requests per second** (throughput).

## Credits

Each API request costs a fixed number of credits based on the endpoint's
computational cost. Credits reset **on the 1st of every month** at midnight
UTC.

| Plan | Monthly Credits | Overage Cost |
|------|----------------|-------------|
| Free | 500 | $2.00 / 1K |
| Starter | 10,000 | $1.00 / 1K |
| Pro | 100,000 | $0.50 / 1K |
| Enterprise | Custom | $0.25 / 1K |

### Credit Consumption by Endpoint

| Endpoint | Credits |
|----------|---------|
| `POST /v1/convert/pdf-to-docx` | 5 |
| `POST /v1/convert/image-to-webp` | 2 |
| `POST /v1/screenshot` | 3 |
| `GET /v1/fuel/prices` | 2 |
| `GET /v1/account/*` | 0 |

## Rate Limits (Requests/Second)

| Plan | Requests/Second | Burst |
|------|----------------|-------|
| Free | 5 | 10 |
| Starter | 50 | 100 |
| Pro | 200 | 500 |
| Enterprise | 1,000 | 2,000 |

## Response Headers

Every response includes usage information in the headers:

| Header | Description |
|--------|-------------|
| `X-Remaining-Credits` | Credits remaining this billing period |
| `Retry-After` | Seconds to wait before retrying (on 429 only) |

## What happens when I run out of credits?

You will receive a `402 Payment Required` error. Your API keys remain active,
and new credits are granted on the 1st of the next month. You can also
purchase overage or upgrade your plan from the dashboard.
